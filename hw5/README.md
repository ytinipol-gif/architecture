## Общая информация
Лабораторная работа выполнена как продолжение ЛР4.  
В предыдущей работе был реализован REST API для системы управления событиями на базе MongoDB.  
В рамках ЛР5 в этот проект добавлены механизмы оптимизации производительности:

- кеширование часто запрашиваемых данных;
- инвалидация кеша после изменения данных;
- ограничение частоты запросов для отдельных endpoints.

В качестве СУБД используется MongoDB, для кеширования и хранения счетчиков rate limiting используется Redis, API реализован на FastAPI.

## Реализованная функциональность
В проекте добавлено кеширование для следующих запросов:

- `GET /api/v1/events`
- `GET /api/v1/events/by-date`
- `GET /api/v1/events/{event_id}/participants`
- `GET /api/v1/users/me/events`

При обработке кешируемых запросов в ответе используется заголовок `X-Cache`:

- `X-Cache: MISS` — данные получены из базы данных;
- `X-Cache: HIT` — данные получены из кеша.

Rate limiting реализован для следующих endpoints:

- `POST /api/v1/auth/login`
- `GET /api/v1/users/search`

При превышении допустимого количества запросов API возвращает код `429 Too Many Requests`.  
Дополнительно в ответ включаются заголовки:

- `X-RateLimit-Limit`
- `X-RateLimit-Remaining`
- `X-RateLimit-Reset`

## Структура проекта
- `performance_design.md` — описание анализа производительности, стратегии кеширования и rate limiting;
- `app/` — исходный код приложения;
- `tests/` — автоматические тесты;
- `data.js` — заполнение базы тестовыми данными;
- `validation.js` — настройка валидации коллекции `events`;
- `queries.js` — набор запросов для проверки операций с данными;
- `Dockerfile` — сборка контейнера с API;
- `docker-compose.yml` — запуск MongoDB, Redis и API.

## Порядок запуска

1. Перейти в каталог проекта:
cd ...hw5"

2. Запустить контейнеры:
docker compose up --build -d

3. Загрузить тестовые данные:
docker compose exec mongo mongosh --file /scripts/data.js

4. Подключить валидацию:
docker compose exec mongo mongosh --file /scripts/validation.js

5. При необходимости выполнить подготовленные запросы:
docker compose exec mongo mongosh --file /scripts/queries.js

После запуска доступны:

- API: `http://localhost:8002`
- Swagger UI: `http://localhost:8002/docs`
- MongoDB: `mongodb://localhost:27018`
- Redis: `localhost:6380`

Для ручной проверки можно открыть Swagger UI и выполнить следующие действия:

1. зарегистрировать пользователя через `POST /api/v1/auth/register`;
2. выполнить вход через `POST /api/v1/auth/login`;
3. создать событие через `POST /api/v1/events`;
4. дважды вызвать `GET /api/v1/events`.

При первом обращении к `GET /api/v1/events` ожидается `X-Cache: MISS`, при повторном обращении — `X-Cache: HIT`.

Для проверки rate limiting можно несколько раз подряд выполнить `POST /api/v1/auth/login`.  
После превышения лимита сервис должен вернуть `429 Too Many Requests`.
